# OTC 订单详情页重排设计

## 背景

当前 `mscoin-frontend/src/pages-vue3/otc/Chat.vue` 与 `mscoin-frontend/src/pages-vue3/otc/Chatline.vue` 仍保留较重的旧式双栏聊天页结构。订单状态、倒计时、支付信息、主操作按钮、聊天记录和输入区同时堆叠，视觉主次不清，无法体现“订单工作台”的核心目标。

本次只重排 `#/otc/chat/:id` 页面，不改订单状态机、接口协议和聊天消息读写逻辑。

## 目标

将 OTC 订单详情页调整为“订单工作台”布局：

- 页面顶部优先展示订单状态、倒计时、订单号和主操作按钮。
- 左侧承载订单信息、支付信息、操作说明。
- 右侧承载聊天记录与消息输入。
- 保持当前接口和交互逻辑不变，只调整信息组织和视觉层级。

## 布局方案

### 1. 顶部状态工作条

页面主区域顶部新增单独状态条，集中放置：

- 订单状态文案
- 倒计时
- 订单号
- 主操作按钮组

状态条是整页最强视觉锚点，用于把“待付款 / 待放行 / 已完成 / 已取消 / 申诉中”的状态和可执行动作第一时间暴露出来。

### 1.1 面包屑路径修正

当前页面面包屑固定显示为 `我的订单 >> 订单详情`，与“用户刚从确认购买成功跳入订单详情”的实际进入路径不一致。

本次面包屑需要按进入上下文动态呈现：

- 从 OTC 下单成功进入时：
  - 显示 `OTC交易 >> 订单详情`
- 从用户中心我的订单进入时：
  - 显示 `我的订单 >> 订单详情`

如果当前代码无法可靠获取来源页，则默认优先使用 `OTC交易 >> 订单详情`，因为该页面当前最核心的入口是下单成功后的订单跟进。

### 2. 双栏工作台主体

顶部状态条下方改成两栏：

- 左栏：订单信息与支付信息
- 右栏：聊天区

左栏宽度固定，右栏自适应。桌面端优先保证聊天区有足够高度；移动端退化为单列堆叠。

### 3. 左栏卡片拆分

左栏拆成三个卡片：

- `订单概览卡`
  - 交易对象
  - 单价
  - 数量
  - 总额
  - 下单时间 / 支付时限
- `收款方式卡`
  - 银行卡
  - 支付宝
  - 微信
  - 二维码入口
- `操作说明卡`
  - 买家/卖家操作说明
  - 风险提示

不再把支付信息横向挤成三列，改为纵向卡片列表，避免信息拥堵。

### 4. 右栏聊天卡

右栏做成单个聊天主卡片：

- 顶部：聊天标题、系统提示、历史消息加载入口
- 中部：消息滚动区
- 底部：输入框、发送按钮、消息提醒开关

输入区固定在聊天卡底部，聊天滚动区高度自适应，确保“查看消息”和“继续沟通”是连续动作。

## 组件结构调整

### Chat.vue

负责：

- 页面级布局容器
- 状态条
- 左栏三个信息卡
- 对 `Chatline.vue` 传递聊天和订单上下文
- 弹窗保留原逻辑，但统一挂在工作台容器下

### Chatline.vue

负责：

- 聊天记录滚动区
- 历史记录加载
- 输入和发送
- 提醒开关

不再承担页面级边框和大块外层定位，改为内聚的聊天卡内容组件。

## 数据与交互

本次不改以下逻辑：

- `/otc/order/detail`
- `/otc/order/pay`
- `/otc/order/cancel`
- `/otc/order/appeal`
- `/otc/order/release`
- socket 消息收发
- 订单状态映射

只调整这些逻辑在页面中的呈现位置，以及页面导航文案的呈现方式。

## 视觉方向

- 延续 OTC 已收口的深色交易风格
- 顶部状态条使用更强对比度
- 卡片统一圆角、边框、背景层级
- 左栏信息以标签-数值对齐方式呈现
- 右栏聊天保留会话感，但弱化“纯聊天产品”视觉

## 测试与验收

实现后至少验证：

- 订单状态条是否正确展示状态和按钮
- 面包屑是否能正确反映进入上下文
- 左栏信息卡是否完整承载订单与支付信息
- 聊天消息滚动、发送、历史加载是否正常
- 买家与卖家两种视角下布局是否都成立
- 桌面端与移动端是否能正常降级

## 范围边界

本次不处理：

- 聊天气泡文案优化
- 弹窗文案重写
- 订单流程逻辑改造
- 新增接口或字段
