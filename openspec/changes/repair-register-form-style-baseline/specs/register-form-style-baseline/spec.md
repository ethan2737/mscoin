## ADDED Requirements

### Requirement: 注册页表单输入框统一暗色样式基线

注册页所有输入框和选择框应采用与登录页一致的暗色主题样式，包括背景色、描边、hover 态、focus 态和 error 态。

#### Scenario: 输入框默认状态
- **WHEN** 用户打开注册页且未与输入框交互
- **THEN** 所有输入框背景色为 `#17212e`，边框色为 `#27313e`，文字颜色为 `#fff`

#### Scenario: 输入框 hover 状态
- **WHEN** 用户鼠标悬停在输入框上
- **THEN** 输入框边框色保持 `#27313e`，无视觉变化或轻微高亮

#### Scenario: 输入框 focus 状态
- **WHEN** 用户点击输入框开始输入
- **THEN** 输入框内阴影为 `inset 0 0 0 1px #f0ac19`，边框色不变

#### Scenario: 输入框 error 状态
- **WHEN** 表单校验失败
- **THEN** 输入框边框色变为红色系，显示错误提示

### Requirement: 区号选择框与输入框视觉一体化

手机号输入行的区号选择框（prepend）与右侧输入框应呈现为统一的视觉整体，无割裂感。

#### Scenario: 区号框默认状态
- **WHEN** 用户未与手机号输入行交互
- **THEN** 区号选择框与右侧输入框背景色一致，均为 `#17212e`，边框无缝衔接

#### Scenario: 区号框 focus 状态
- **WHEN** 用户点击区号下拉框或输入框
- **THEN** 整个手机号输入行（含区号和号码）统一进入 focus 状态，边框同时变色

### Requirement: 邀请码输入框 prepend 文本样式优化

邀请码输入框的 prepend 文本区域应与输入区形成一体化控件，避免视觉突兀。

#### Scenario: 邀请码框默认状态
- **WHEN** 用户未与邀请码输入框交互
- **THEN** prepend 文本区域与输入区背景色一致，无额外边框或色块

### Requirement: 验证码行按钮暗色样式

"发送验证码"按钮应采用暗色次级按钮风格，与深色卡片主题协调。

#### Scenario: 验证码按钮默认状态
- **WHEN** 用户未与验证码按钮交互
- **THEN** 按钮背景色为 `#27313e`，文字颜色为 `#fff`，高度与输入框一致

#### Scenario: 验证码按钮 hover 状态
- **WHEN** 用户鼠标悬停在验证码按钮上
- **THEN** 按钮文字颜色变为 `#f0a70a`，背景色不变或轻微变亮

#### Scenario: 验证码按钮 disabled 状态
- **WHEN** 验证码进入倒计时期间
- **THEN** 按钮背景色变暗，文字颜色变灰，不可点击

### Requirement: 注册页垂直居中卡片布局

注册页主卡片应采用垂直居中布局，与登录页视觉重心对齐。

#### Scenario: 桌面端布局
- **WHEN** 用户在桌面端访问注册页
- **THEN** 注册卡片在视口中垂直居中，奖励跑马灯（如有）作为卡片上方独立模块

#### Scenario: 移动端布局
- **WHEN** 用户在移动端访问注册页
- **THEN** 保持原有流式布局，卡片靠上显示

### Requirement: 协议勾选区与人机验证区交互优化

协议勾选框和人机验证按钮应增强交互反馈和视觉层级。

#### Scenario: 协议链接 hover 状态
- **WHEN** 用户鼠标悬停在《用户协议》链接上
- **THEN** 链接文字颜色变为 `#f0a70a`，显示下划线

#### Scenario: 人机验证完成状态
- **WHEN** 用户完成人机验证
- **THEN** 验证区边框或背景色变化，显示"已完成"视觉提示
