# 业务域现状梳理

更新时间：2026-04-09

## 1. 目的

本文档只从业务流程角度梳理当前项目状态，用来回答两个问题：

- 文档中定义了哪些业务域
- 在当前升级后的开发版本里，哪些业务已经跑通，哪些还没有真正触达

这里不讨论技术实现细节，只看业务是否已经能够被实际走通或验证。

## 2. 结论摘要

当前版本可以认为：

- `现货主流程`：已跑通
- `内容浏览类流程`：已跑通
- `合约 / OTC / 挖矿 / 众筹`：已触达，但不是完整真实业务闭环
- `提现 / C2C / 秒合约`：文档中提到，但当前升级版本没有完成业务闭环验证

因此，当前项目状态不是“全业务域全部拉通”，而是：

- 核心交易主流程已经跑通
- 站内部分业务域已经可以访问和验收
- 仍有一部分文档中的业务域，在当前版本下还没有真正完成业务级触达

## 3. 文档中的主要业务域

根据 `docs` 目录现有文档，项目业务范围主要包括：

1. 用户登录 / 注册
2. 首页行情与币种浏览
3. K 线与实时行情
4. 现货交易
5. 委托订单
6. 撮合交易
7. 交易完成
8. 用户钱包
9. 公告 / 帮助内容
10. OTC / C2C
11. 合约 / 永续合约
12. 挖矿 / 众筹类活动
13. 提现
14. 秒合约

## 4. 当前版本已跑通的业务域

以下业务在当前升级版本下，已经能按业务流程实际走通或完成验证。

### 4.1 登录与基础访问

- 用户登录
- 进入首页
- 浏览站内帮助与公告内容

业务判断：

- 这部分已经可以正常使用
- 说明基础用户访问链路已经恢复

### 4.2 首页行情与现货入口

- 首页查看币种行情
- 从首页进入现货交易页

业务判断：

- 首页核心展示能力已恢复
- 用户可以从首页进入交易主流程

### 4.3 现货交易主流程

当前已经确认可以走通的现货主流程：

1. 登录
2. 查看行情
3. 进入现货交易
4. 提交委托
5. 查看当前委托
6. 查看历史委托
7. 验证撤单后的终态结果

业务判断：

- 这是当前版本最明确跑通的一条核心主流程
- 如果只看交易平台最核心的业务链路，可以认为已经打通

### 4.4 公告与帮助内容浏览

- 帮助列表可访问
- 公告详情可访问

业务判断：

- 内容类页面已经恢复到可正常浏览状态
- 属于当前版本已跑通的非交易核心流程

## 5. 当前版本已触达，但还不能视为完整跑通的业务域

这些业务域在当前版本下已经能进入页面、能看到基础内容、或者能做页面级验证，但还不能等同于“完整真实业务闭环已经跑通”。

### 5.1 合约交易

当前状态：

- 已能进入合约页面
- 已能进入合约相关用户中心页面
- 已完成页面级验收

但不能直接下结论为“完整跑通”的原因：

- 当前更多是页面和访问能力恢复
- 不是完整真实合约业务链路的最终验收结果

业务判断：

- `已触达`
- `未完成完整业务闭环验证`

### 5.2 OTC

当前状态：

- 已能进入 OTC 页面
- 已能查看 OTC 交易页面

但当前还不能说明：

- 真实 OTC 交易流程已经全量跑通
- OTC 完整业务闭环已经验收完成

业务判断：

- `已触达`
- `未完成完整业务闭环验证`

### 5.3 挖矿

当前状态：

- 挖矿相关页面可以访问
- 页面级可验证

业务判断：

- `已触达`
- `未完成完整业务闭环验证`

### 5.4 众筹

当前状态：

- 众筹页面可以访问
- 页面级可验证

业务判断：

- `已触达`
- `未完成完整业务闭环验证`

### 5.5 用户中心的部分扩展页面

例如：

- 创新挖矿
- 部分合约委托页

当前状态：

- 页面可进入

- `未完成完整业务闭环验证`

## 6. 文档中提到，但当前升级版本还没有真正触达或没有完成业务级验证的业务域

以下内容在文档中明确出现，但在当前升级版本下，还不能说已经真正拉通。

### 6.1 C2C

文档中有明确介绍，但当前没有确认过完整业务流程。

- `当前版本未完成业务级触达`

### 6.2 提现

文档中有完整提现流程说明，但当前并没有把“提现业务闭环”作为已跑通结论的一部分。

业务判断：

- `文档已定义`
- `当前版本未完成业务级验证`

### 6.3 秒合约

项目介绍中提到秒合约，但当前没有完成这条业务流的验收。

业务判断：

- `文档已定义`
- `当前版本未完成业务级触达`

### 6.4 现货成交完成态的完整业务闭环

虽然现货主流程已经跑通，但文档中的“撮合交易 -> 交易完成 -> 钱包变化”是一条更完整的业务闭环。

当前已经确认的是：

- 下单
- 查看委托
- 撤单终态

当前还不能直接等同于：

- 已完成真实成交后的完整终态业务验收

业务判断：

- `主流程已跑通`
- `文档中的完整成交闭环尚未完全对齐`

## 7. 对齐结论

如果只从业务角度看，当前项目和 `docs` 的关系可以分成三层：

### 第一层：已经对齐

- 登录
- 首页行情
- 现货主流程
- 公告 / 帮助浏览

### 第二层：部分对齐

- 合约
- OTC
- 挖矿
- 众筹
- 部分用户中心扩展业务

这些业务已经在当前版本中出现并可访问，但还没有达到“文档级完整业务闭环”的程度。

### 第三层：尚未对齐

- C2C
- 提现
- 秒合约
- 现货完整成交完成态

这些业务要么文档中已有定义，但当前版本没有真正完成触达；要么只完成了主流程的一部分，没有完成完整业务验证。

## 8. 最终判断

当前版本可以明确认定：

- 核心业务已经跑通
- 但全业务域没有全部对齐文档

更准确地说：

- 当前项目已经恢复到“可以继续做业务测试和产品迭代”的状态
- 但还不能说“文档中所有业务域都已经在升级版本里完整落地并全部验收完成”

## 9. 建议后续梳理顺序

如果后续继续从业务角度推进，建议按这个顺序补齐：

1. 现货真实成交完成态闭环
2. 合约完整业务闭环
3. OTC / C2C 业务闭环
4. 提现业务闭环
5. 秒合约是否继续保留及其业务边界确认
