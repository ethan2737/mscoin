# 公益创新室业务逻辑梳理

## 1. 功能概述

公益创新室（Innovation Room）是 MSCOIN 平台为用户提供的一个创新活动参与平台，用户可以通过该平台参与平台发行的各类创新活动，包括代币发行、云矿机认购等。该功能将公益活动与金融创新相结合，让用户在参与活动的同时支持公益事业。

### 1.1 主要作用

- **创新活动发行平台**：提供多种类型的活动发行模式，包括首次上线（抢购/分摊）、持仓瓜分、自由认购、云矿机认购等
- **用户参与入口**：为用户提供统一的活动浏览、详情查看、参与申购的入口
- **资产管理**：用户可查看自己参与的活动订单和持有的矿机资产
- **公益结合**：将部分活动收益用于公益事业，实现"数字金融 + 众筹 = 永续公益"的理念

### 1.2 子功能模块

| 子功能 | 路由 | 说明 |
|--------|------|------|
| 活动列表页 | `/activity` | 展示所有活动，支持按状态筛选（全部/准备中/进行中/派发中/已完成） |
| 活动详情页 | `/activity/:id` | 展示活动详细信息，支持用户参与申购 |
| 我的订单 | `/uc/innovation/myorders` | 查看用户参与的活动订单记录 |
| 我的矿机 | `/uc/innovation/myminings` | 查看用户持有的云矿机及挖矿收益 |

## 2. 用户操作流程

### 2.1 浏览活动列表

**操作步骤：**

1. 用户通过顶部导航"活动中心"或首页广告轮播进入活动列表页
2. 用户可切换标签查看不同状态的活动：
   - `全部` - 显示所有活动
   - `准备中` - 即将开始的活动
   - `进行中` - 正在进行的活动
   - `派发中` - 正在派发 Token 的活动
   - `已完成` - 已结束的活动
3. 用户点击"查看详情"按钮进入活动详情页

### 2.2 查看活动详情

**操作步骤：**

1. 用户进入活动详情页后，可查看活动的基本信息：
   - 活动名称、封面图、简介
   - 活动进度条（已兑数量/总量）
   - 活动状态（准备中/进行中/派发中/已完成）
   - 活动类型说明
   - 发行价格、总量、限购规则
   - 开始/结束时间
   - 活动说明和规则
2. 对于自由认购和云矿机类型活动，页面显示已兑/剩余数量
3. 对于持仓瓜分类型活动，页面显示用户当前持仓可瓜分数量
4. 用户可在页面右侧看到生成活动分享的二维码

### 2.3 参与活动申购

**操作步骤：**

1. 用户登录账号后，在进行中的自由认购/持仓瓜分/云矿机活动中，可输入申购数量
2. 系统显示用户当前可用余额
3. 用户点击"立即参与"按钮
4. 弹出安全验证窗口，根据用户绑定的手机号或邮箱发送验证码
5. 用户输入验证码并提交
6. 系统验证通过后，冻结相应资产，申购成功

### 2.4 查看我的订单

**操作步骤：**

1. 用户登录账号后，进入"个人中心"
2. 点击"活动中心" → "我的订单"
3. 查看已参与的活动订单列表，包括：
   - 活动名称、类型
   - 申购数量、参与币种
   - 订单状态（待成交/已成交/已撤销）
   - 成交额、创建时间

### 2.5 管理我的矿机

**操作步骤：**

1. 用户登录账号后，进入"个人中心"
2. 点击"活动中心" → "我的矿机"
3. 查看持有的云矿机列表，包括：
   - 矿机名称、状态（待部署/已部署/已结束）
   - 产出币种、产出周期
   - 基础产能、实际产能（含邀请加成）
   - 挖矿周期、已挖周期
   - 总产出收益
4. 当有可领取的收益时，点击"待领取"按钮领取收益
5. 点击"邀请"按钮可查看邀请好友提升产能的规则

## 3. 业务逻辑说明

### 3.1 活动列表数据流转

```
用户访问活动列表页
  ↓
前端请求 POST /uc/activity/page-query
  ↓
Vite 代理请求到 ucenter-api:8888
  ↓
ucenter-api 的 ActivityHandler 接收请求
  ↓
ActivityLogic.ActivityPageQuery 处理业务逻辑
  ↓
从 activitySeeds 种子数据中筛选并分页
  ↓
返回活动列表数据
  ↓
前端渲染活动卡片列表
```

**前端代码位置：** `mscoin-frontend/src/pages-vue3/activity/Activity.vue`

**后端代码位置：** `mscoin-backend/ucenter-api/internal/logic/activity.go`

**API 接口：** `POST /uc/activity/page-query`

### 3.2 活动详情数据流转

```
用户访问活动详情页
  ↓
前端请求 POST /uc/activity/detail?id={activityId}
  ↓
ucenter-api 的 ActivityHandler 接收请求
  ↓
ActivityLogic.ActivityDetail 查询活动详情
  ↓
从 activitySeeds 中查找匹配的活动
  ↓
返回活动详细信息
  ↓
前端渲染详情页内容
  ↓
同时调用 GET /uc/wallet/{acceptUnit} 获取用户余额（已登录）
  ↓
调用 POST /uc/member/activity 获取用户已参与记录（已登录）
  ↓
调用 POST /uc/approve/security/setting 获取用户会员等级
```

**前端代码位置：** `mscoin-frontend/src/pages-vue3/activity/ActivityDetail.vue`

**API 接口：**
- `POST /uc/activity/detail`
- `POST /uc/wallet/{unit}`
- `POST /uc/member/activity`
- `POST /uc/approve/security/setting`

### 3.3 参与活动申购数据流转

```
用户点击"立即参与"
  ↓
前端校验登录状态
  ↓
校验申购数量（最小起兑量、最大限购量、限购次数）
  ↓
弹出安全验证窗口
  ↓
前端请求 POST /uc/mobile/code 发送手机验证码
  或 POST /uc/activity/code 发送邮箱验证码
  ↓
用户输入验证码并提交
  ↓
前端请求 POST /uc/activity/attend
  ↓
ActivityLogic.ActivityAttend 处理申购
  ↓
校验用户资产余额
  ↓
冻结用户资产
  ↓
创建活动订单记录
  ↓
返回申购结果
  ↓
前端提示申购成功并刷新页面
```

**API 接口：** `POST /uc/activity/attend`

### 3.4 我的订单数据流转

```
用户访问"我的订单"页面
  ↓
前端请求 POST /uc/activity/myOrders
  ↓
后端查询用户参与的活动订单
  ↓
返回订单列表数据
  ↓
前端渲染订单表格
  ↓
用户点击订单链接跳转到活动详情页
```

**前端代码位置：** `mscoin-frontend/src/pages-vue3/uc/InnovationOrders.vue`

**API 接口：** `POST /uc/activity/myOrders`

### 3.5 我的矿机数据流转

```
用户访问"我的矿机"页面
  ↓
前端请求 POST /uc/miningorder/my-minings
  ↓
后端查询用户持有的云矿机
  ↓
返回矿机列表数据
  ↓
前端渲染矿机卡片列表
  ↓
用户点击"待领取"按钮
  ↓
前端请求 POST /uc/miningorder/fetch-profit
  ↓
后端处理收益领取
  ↓
返回领取结果
  ↓
前端刷新矿机列表
```

**前端代码位置：** `mscoin-frontend/src/pages-vue3/uc/InnovationMinings.vue`

**API 接口：**
- `POST /uc/miningorder/my-minings`
- `POST /uc/miningorder/fetch-profit`

## 4. 活动类型说明

### 4.1 首次上线（抢购）- Type 1

- **特点**：限时限量抢购，先到先得
- **状态流转**：准备中 → 进行中 → 派发中 → 已完成
- **限购规则**：支持单笔最小/最大数量限制、参与次数限制
- **资产处理**：参与时冻结资产，活动结束后统一分配

### 4.2 首次上线（分摊）- Type 2

- **特点**：按参与比例分摊，人人有份
- **状态流转**：准备中 → 进行中 → 派发中 → 已完成
- **分配规则**：活动结束后按参与比例分配总量
- **资产处理**：参与时冻结资产，活动结束后按比例分配

### 4.3 持仓瓜分 - Type 3

- **特点**：按持仓比例瓜分活动总量
- **状态流转**：准备中 → 进行中 → 派发中 → 已完成
- **参与要求**：需要持有指定币种
- **分配公式**：用户可瓜分数 = (用户持仓 / 总持仓) × 活动总量
- **资产处理**：参与活动需锁仓至活动结束

### 4.4 自由认购 - Type 4

- **特点**：自由申购，多买多送
- **状态流转**：准备中 → 进行中 → 派发中 → 已完成
- **限购规则**：支持单笔最小/最大数量限制、参与次数限制
- **会员折扣**：支持不同会员等级享受不同折扣
- **产能加成**：支持邀请好友获得产能提升

### 4.5 云矿机认购 - Type 5

- **特点**：购买云矿机获得持续产币收益
- **状态流转**：准备中 → 进行中 → 已结束
- **产出周期**：支持按天/周/月/年产出
- **产能计算**：基础产能 + 邀请加成（每邀请一人提升固定百分比）
- **收益领取**：用户可手动领取待领取收益

## 5. 数据结构定义

### 5.1 活动数据结构

```go
type Activity struct {
    Id             int64   // 活动 ID
    Title          string  // 活动标题
    Detail         string  // 活动简介
    SmallImageUrl  string  // 封面图片 URL
    Type           int64   // 活动类型 1-5
    Step           int64   // 活动状态 0-3
    StartTime      int64   // 开始时间戳（毫秒）
    EndTime        int64   // 结束时间戳（毫秒）
    TotalSupply    float64 // 活动总量
    TradedAmount   float64 // 已兑数量
    Progress       float64 // 进度百分比
    Price          float64 // 价格
    PriceScale     int     // 价格精度
    Unit           string  // 活动货币单位
    AcceptUnit     string  // 接受货币单位
    AmountScale    int     // 数量精度
    MaxLimitAmout  float64 // 最大限购量
    MinLimitAmout  float64 // 最小限购量
    LimitTimes     int     // 限购次数
}
```

### 5.2 矿机数据结构（前端展示）

```javascript
{
    activityId: 1,           // 活动 ID
    title: "云矿机",          // 矿机名称
    image: "...",            // 矿机图片
    miningStatus: 0/1/2,     // 挖矿状态 0=待部署 1=已部署 2=已结束
    miningUnit: "KICK",      // 产出币种
    totalProfit: 100,        // 总产出收益
    miningTimes: 365,        // 挖矿周期
    minedgedTimes: 100,      // 已挖周期
    miningDaysprofit: 0.27,  // 基础产能
    currentDaysprofit: 0.35, // 实际产能（含邀请加成）
    miningDays: 365,         // 周期单位
    period: 0/1/2/3,         // 周期类型 0=天 1=周 2=月 3=年
    miningInvite: 0.12,      // 邀请加成比例
    miningInvitelimit: 0,    // 邀请加成上限（0=无上限）
    canFetchProfit: 10       // 可领取收益
}
```

## 6. 状态码说明

### 6.1 活动状态（Step）

| 值 | 状态 | 说明 |
|----|------|------|
| 0 | 准备中 | 活动即将开始，用户可查看但不可参与 |
| 1 | 进行中 | 活动进行中，用户可参与申购 |
| 2 | 派发中 | 活动已结束，正在派发 Token |
| 3 | 已完成 | 活动已完成，Token 已发放 |

### 6.2 矿机状态（MiningStatus）

| 值 | 状态 | 说明 |
|----|------|------|
| 0 | 待部署 | 矿机已购买，等待部署 |
| 1 | 已部署 | 矿机正在运行产币中 |
| 2 | 已结束 | 矿机挖矿周期已结束 |

### 6.3 订单状态（Order State）

**对于矿机订单（Type 5）：**
| 值 | 状态 | 说明 |
|----|------|------|
| 1 | 未部署 | 订单已创建，等待部署 |
| 2 | 已部署 | 矿机已部署运行 |
| 3 | 已撤销 | 订单已撤销 |

**对于其他订单类型：**
| 值 | 状态 | 说明 |
|----|------|------|
| 1 | 待成交 | 等待活动结束成交 |
| 2 | 已成交 | 活动结束已成交 |
| 3 | 已撤销 | 订单已撤销 |

## 9. 注意事项

1. **当前状态**：后端活动数据使用 Mock 种子数据（activitySeeds），未连接真实数据库
2. **登录态要求**：参与活动、查看订单和矿机需要用户登录，使用 `x-auth-token` 请求头传递 Token
3. **安全验证**：参与活动申购需要手机或邮箱验证码验证
4. **资产冻结**：参与活动后资产会被冻结，活动结束后解冻或分配 Token
5. **邀请加成**：云矿机类型支持邀请好友提升产能，产能提升比例和上限由活动配置决定
