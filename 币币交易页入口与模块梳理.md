# 币币交易页入口与模块梳理

本文结论以当前仓库代码和 2026-04-12 本地联调结果为准。

验证依据：

- 前端页面与路由：`mscoin-frontend/src/App.vue`、`mscoin-frontend/src/config/routes-vue3.js`、`mscoin-frontend/src/pages-vue3/exchange/Exchange.vue`
- 后端接口：`mscoin-backend/market-api/internal/handler/routes_handler.go`、`mscoin-backend/exchange-api/internal/handler/routes_handler.go`、`mscoin-backend/exchange/internal/logic/order_logic.go`
- 本地实测：`http://127.0.0.1:3000`、`8888`、`8889`、`8890` 已启动，使用基线账号 `13800000000 / 123456` 对交易页核心接口做了实际调用

## 一、币币交易页所有入口

| 入口 | 代码位置 | 实际路径 | 结论 |
| --- | --- | --- | --- |
| 顶部主导航 | `mscoin-frontend/src/App.vue` | `/exchange` | 桌面端可进入，进入后页面会自动补全到默认交易对 `/exchange/btc_usdt` |
| 首页行情表“去交易”按钮 | `mscoin-frontend/src/pages-vue3/index/Index.vue` | `/exchange/${coin.href}` | 可进入具体交易对，当前是最明确、最稳定的入口 |
| 直接地址访问 | `mscoin-frontend/src/config/routes-vue3.js` | `/exchange`、`/exchange/:pair` | 可直接进入；`/exchange/:pair` 是最完整入口 |
| 页面内切换交易对 | `mscoin-frontend/src/pages-vue3/exchange/Exchange.vue` | `router.push({ name: 'ExchangePair', params: { pair } })` | 可在页内切换到其他交易对 |

当前未发现 Vue 3 活动页、公告页、用户中心对币币交易页有显式跳转入口。

需要单独指出的入口问题：

- 移动端顶部抽屉菜单里没有 `/exchange` 菜单项。
- 同时 `mscoin-frontend/src/App.vue` 中对 `to.path === '/exchange' && isMobile()` 做了 `router.replace('/reg')`，意味着移动端访问裸路由 `/exchange` 会被重定向到注册页。
- 结论：移动端“裸入口 `/exchange`”当前不通，且导航层没有明显替代入口。

## 三、已实测跑通的核心链路

### 1. 页面进入链路

- 访问 `/exchange/BTC_USDT` 可正常进入交易页。
- 页面初始化会依次请求汇率、币对精度、币种信息、交易对列表、盘口、深度、最新成交。

### 2. 委托查询链路

- `/exchange/order/current` 本地返回当前委托。
- `/exchange/order/history` 本地返回历史委托。

### 3. 限价下单 + 撤单链路

- 调用 `/exchange/order/add` 提交 `BTC/USDT` 限价买单，接口返回成功和新订单号。
- 调用 `/exchange/order/cancel` 取消该订单，接口返回成功。
- 取消后该订单不再出现在当前委托中，并在历史委托中显示为 `CANCELED`。

## 四、当前明确存在的问题

### 1. 自选不是后端真链路

- `mscoin-frontend/dev/localAcceptanceMocks.mjs` 明确拦截了：
  - `/exchange/favor/find`
  - `/exchange/favor/add`
  - `/exchange/favor/delete`
- 当前本地“自选可用”是前端开发 mock 结果，不是 `exchange-api` 真实能力。

### 2. 移动端 `/exchange` 入口异常

- 移动端没有交易页菜单入口。
- 裸访问 `/exchange` 会被重定向到 `/reg`。
- 结论：移动端主入口存在明显断点。

### 3. 市价单数据回显不完整

- 市价买单接口返回成功，但历史委托数据里缺少页面依赖的 `status`、`price`、`tradedAmount` 等字段。
- 页面虽然能发起请求，但委托表格未必能完整表达市价单状态。

### 4. 委托详情展开层不可用

- 页面展开层依赖 `row.detail`。
- 当前本地返回的委托记录中未提供 `detail`。
- `mscoin-frontend/src/pages-vue3/exchange/expand.vue` 仍保留旧式过滤器语法，不适合 Vue 3 当前写法。

## 五、最终结论

当前币币交易页主链路整体情况如下：

- 入口方面：桌面端可进，首页入口可进，页内切对可进；移动端裸入口异常。
- 行情方面：交易对列表、头部行情、盘口、最新成交、当前/历史委托都已跑通。
- 交易方面：限价单提交和撤单已跑通。
- 风险点方面：自选是前端 mock，不是后端真链路；市价单回显不完整；委托展开详情未跑通。

如果要按“可上线”标准看，当前币币交易页不是完全跑通状态，至少还需要补齐：

1. 自选真实后端接口
2. 市价委托回显字段
3. 委托详情展开数据
4. 移动端交易页入口
